- 명확하고 독창적인 타이틀의 사용하기
 title 태그를 이용하여 페이지의 제목 나타내기
   title 태그 : 사용자와 검색 엔진에 특정 페이지의 주제를 알려줌
   title 태그는 html 문서의 head 태그 내에 있어야 함
   각각의 페이지마다 고유한 제목을 만드는것이 이상적인 방법. title 태그는 주로 결과의 첫번째 행에 나타나므로 중요함.
   
  title 태그를 위한 권장 사항
   - 페이지의 콘텐츠를 정확하게 설명하는 제목
   - 페이지마다 고유한 title 태그 작성
     사이트 내의 서로 다른 페이지를 구별하는데 도움이 됨
   - 간결하면서 내용을 포함한 제목 작성.
     title 태그는 간단명료하면서 페이지 내용에 대한 정보를 제공할 수 있도록 작성. 너무 길면 검색결과에 일부붐나 표시됨.
   
- "description" 메타 태그 활용하기
  각 페이지의 요약 정보를 보여주기
    페이지에 description 메타 태그를 사용하면 google 및 다른 검색 엔진에서 페이지 정보의 요약 내용을 파악하여 검색결과의 미리보기에 인용할 수 있음
	
  description 태그를 위한 궈장 사항
    - 페이지의 내용을 정확하게 요약하기
	  검색 결과에서 desscription 메타 태그를 보게되므로, 사용자에게 정보를 제공하고 동시에 관심을 유도할 수 있도록 작성
	- 각 페이지 마다 내용에 맞는 고유한 설명 사용
    
	
- 페이지의 URL 구조를 개선하기
 이해하기 쉬운 URL은 콘텐츠의 내용을 전달하는데 좋습니다.
   웹사이트의 문서에 카테고리와 파일 이름을 효과적으로 만들면 사이트를 크롤링하기도 쉬워짐
   URL에 알 수 없는 매개변수가 많이 표시된 경우에, 사용자는 URL의 일부분이 불필요하다고 생각하여, 링크 시 일부분을 빠뜨려 링크가 깨질 수 있음
   URL은 검색결과의 일부로도 보여지게 되므로, 사이트/페이지와 관련있는 URL을 사용하면 좋음
   구글이 권장하는 URL 구조 (https://support.google.com/webmasters/answer/76329)
   
   URL 구조를 위한 권장 사항
   - URL에 단어 사용
   - 단순 디렉토리 구조 만들기
     사이트 전체의 내용을 잘 구성하면서도 방문자가 사이트에서 현재 위치를 잘 알 수 있돌고 해주는 디렉토리 구조를 사용
	 해당 URL에 있는 내용의 유형을 표시하는 디렉토리 구조를 사용하여 해당 URL이 내용 분류가 잘 전달될 수 있도록 함
   - 특정 문서에 도달하기 위한 한가지 형태의 URL 제공
     같은 페이지를 서로 다른 버전의 URL을 가지고 링크하는 경우 해당 내용에 대한 인지도가 여러 URL로 분산될 수 있으므로 페이지 구조 및 내부 링크에 하나의 URL만을 참조하는 것이 좋음
     만약 여러 URL을 통해 같은 내용의 페이지에 접속되는 경우 URL중에서 가장 우선적인 URL로 301 리다이렉션을 설정하는 것이 좋은 해결방법
     만약 리다이렉션을 사용할 수 없다면 rel="canonical" 링크 속성을 이용하여 표준 URL을 지정하도록 함


- 사이트 내에서 이동하기 쉽게 만들기
 검색 엔진에게 사이트 내 이동은 매우 중요
   이는 각각의 페이지가 전체 사이트내에서 어떤 역할을 담당하는지를 파악하는데 도움이 됨
 홈페이지를 기반으로 한 이동 경로 계획하기
   방문자가 첫 페이지(루트 페이지)에서 더 구체적인 콘텐츠가 있는 페이지로 이동하는 방법에 대해 생각해야 함
   여러 카테고리 및 하위 카테고리 페이지로 나누어야 할 정도로 주제를 세분화해야 하는지를 확인한 후, 각 주제로의 이동 경로를 계획
 '사이트 이동 경로' 링크를 사용하므로 방문자에게 편리를 제공
 사용자가 URL의 일부를 제거하는 경우 발생할 상황을 고려
 실제 사용자와 검색엔진을 위한 두 종류의 사이트맵 준비
 
 사이트 내 쉬운 이동을 위한 권장사항
  - 자연스러운 계층 구조 만들기
  - 이동 경로를 위해 텍스트링크를 사용
  - 사이트에 HTML 사이트맵 페이지 배치 및 XML 사이트맵 파일 사용
  - 유용한 404 페이지 사용

- 이미지 사용의 최적화
 alt 속성을 이용하여 이미지를 표시할 수 없는 경우 대체 텍스트가 보여지도록 설정(스크린 리더를 사용할 경우 대체 텍스트로 읽음)

- 제목 태그의 적절한 활용
 권장 사항
  - 제목 태그는 내용의 개요를 작성하는 작업
  - 한 페이지안에 제목 태그를 신중하게 사용
    제목 태그는 필요한 경우에만 사용. 한 페이지에 제목 태그가 너무 많으면 사용자가 중요한 콘텐츠를 파악하기 어렵고, 각각 다른 주제를 구분하기 어려울 수 있음
	
	
- robot.txt를 효과적으로 활용하기
 "robot.txt" 파일은 검색 엔진이 사이트에 접근하고 크롤링 수 있는 부분을 알려주어 검색 엔진의 트레픽을 조절하는데 사용됨
 사이트에 하위 도메인이 있고 특정 하위 도메인에 있는 특정 페이지가 크롤링되지 않기를 바라는 경우 해당 하위 도메인에 대해 별도의 robot.txt 파일을 만들어야 함
 "Fetch As Google" 기능을 통하여 페이지의 크롤링 상태를 확인하고 문제가 있는 페이지를 확인할 수 있다
 
 권장 사항
  - robots.txt를 보안 도구로 사용하지 않기
    robots.txt는 정상적인 검색 엔진의 트래픽을 조절하기 위한 규약이다. 과도한 robot.txt 설정은 사이트에 더 많은 방문자를 유도할 수 있는 정상적인 검색 엔진만을 차단할 뿐이다. 악성코드를 유발하는 수많은 악의적인 로봇들은 이러한 로봇 배제 표준을 따르지 않으며 많은 검색엔진들이 제공하는 좋은 정보를 제공받지 못하게 되어 악성로봇의 공격에 노출되어도 쉽게 파악할 수 없게 된다.
  - 민감한 콘텐츠는 보다 안전한 방법 사용하기
    민감한 콘텐츠는 암호화 하거나 .htaccess를 사용하여 비밀번호로 보호하는 것이 보다 안전한 방법이다.
  - 웹마스터를 위한 무료 도구 사용하기
  
  
- rel="nofollow" 링크 속성 사용시 유의
 링크의 "rel" 속성 값을 "nofollow"로 설정하면 사이트의 특정 링크를 따르지 말거나 페이지의 인지도를 링크된 페이지로 전달하지 말라는 의미입니다.
